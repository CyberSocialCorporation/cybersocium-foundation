# Contributing to CyberSocium Foundation

Thank you for your interest in contributing to the CyberSocium Foundation paper.

## Types of Contributions

### 1. Error Corrections
- Typos, formatting issues, broken references
- Mathematical errors in proofs or formulas
- Factual inaccuracies in literature review

### 2. Content Improvements
- Additional references to relevant literature
- Improved clarity of explanations
- Translation improvements (RU/EN)

### 3. Theoretical Extensions
- New proofs or theorems
- Formal verification of existing claims
- Comparative analysis with new DAO systems

### 4. Empirical Data
- Simulation results using cadCAD or similar frameworks
- Analysis of on-chain data from GyberExperiment
- Comparative metrics from other DAO ecosystems

## Process

1. **Open an Issue** describing the proposed change
2. **Fork** the repository
3. **Create a branch** from `main`
4. **Make changes** to both RU and EN versions if applicable
5. **Submit a Pull Request** with a clear description

## Style Guidelines

- **Academic register**: maintain scholarly tone throughout
- **Citations**: use [Author, Year] format
- **Technical terms**: preserve untranslated (CSC, PMIP, SES, etc.)
- **Section numbering**: do not change existing numbering without discussion
- **Version**: increment minor version (2.x) for content changes

## Code of Conduct

All contributors are expected to adhere to principles of academic integrity, respectful discourse, and collaborative knowledge production — values that are core to the CyberSocium framework itself.
